# Engwolf

**Engwolf** - это Android-приложение для изучения английского языка с использованием алгоритма интервальных повторений. Проект построен на архитектуре **MVVM** (Model-View-ViewModel) с использованием Jetpack Compose и Room Persistence Library

## Слой данных

### WordEntity
Основная сущность для хранения слов и системных данных
*   **id**: Первичный ключ
*   **word**: Слово или ключ системного параметра
*   **translation**: Перевод слова или значение параметра
*   **dictionaryName**: Категория (уровень "A1", "A2", "B1") или системный раздел "grammar_internal_stats"
*   **intervalStep**: Стадия интервала (0-7) или 999, если слово выучено
*   **nextReviewTime**: Временная метка следующего повторения в миллисекундах

### WordRepository
Центральный компонент логики:
*   **Управление интервалами**: Реализован список `intervals`, определяющий прогрессию времени повторения (от 1 минуты до 30 дней)
*   **Логика SRS**: Метод `updateWordProgress` автоматически меняет `intervalStep` и `nextReviewTime` в зависимости от корректности ответа пользователя
*   **Системные функции**: Работа с JSON-ресурсами для инициализации словарей и сохранение прогресса тестов по грамматике через фиктивные записи в базе данных

### WordDao & AppDatabase
*   Интерфейс DAO обеспечивает асинхронное взаимодействие с базой данных через `suspend` функции
*   Реализованы фильтры для выборки новых слов и слов, ожидающих повторения (`nextReviewTime <= currentTime`)

## Слой представления

### Навигация
*   Используется `NavHost` с маршрутизацией: `first`, `selection`, `quiz/{lvl}`, `grammar`, `grammar_quiz/{sectionId}`, `stat`
*   Реализованы методы `navigateSafe` и `popBackStackSafe` для предотвращения ошибок при быстрой навигации

### QuizViewModel
Управляет состоянием экрана обучения (`QuizUiState`):
*   **Таймер (Ticker)**: Периодически проверяет базу данных на наличие новых слов для изучения
*   **Алгоритм показа**: Приоритезирует повторение старых слов, перемежая их с новыми в рамках лимита
*   **Валидация**: Сравнивает ввод пользователя с корректным переводом

## Алгоритмы и ограничения

### Алгоритм интервальных повторений
1.  **Неверный ответ**: Сброс `intervalStep` в 0, повторение немедленно
2. **Верный ответ**:
   * **Изучаемое слово**: Инкремент `intervalStep` и расчет `nextReviewTime` согласно массиву `intervals`
   * **Завершение**: При достижении финального шага `intervalStep` устанавливается в `999` (изучено)

### Ограничения
*   **Суточный лимит**: Метод `canLearnMore` блокирует изучение новых слов, если пользователь уже выучил 10 новых слов за текущие сутки

## Статистика грамматики
Для хранения прогресса тестов (количество правильных ответов и текущий вопрос) используется таблица `words` с системным словарем `grammar_internal_stats`
*   Ключи `[ID]_score` хранят накопленные баллы
*   Ключи `[ID]_current_q` позволяют возобновить прерванный тест с нужного вопроса

## Внешние ресурсы
Приложение имеет следующие файлы в директории `assets/`:
*   `A1.json`, `A2.json`, `B1.json`: Списки слов (массив объектов `word` и `translation`)
*   `grammar.json`: Список объектов с вопросами, вариантами ответов и индексом правильного ответа
